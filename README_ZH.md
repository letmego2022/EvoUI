# EvoUI – 自进化智能 UI 自动化

**EvoUI** 是一个先进的 UI 自动化框架，结合了 AI 驱动的操作记忆、视觉兜底执行以及智能步骤验证。系统能够从历史操作中学习，复用已有操作记忆，并自适应动态变化的 Web 界面。  

---

## 功能特点

### 1. AI 驱动的操作记忆
- 可以复用之前的 UI 操作以完成新的步骤，高置信度选择最佳动作。
- 支持参数化复用（`reuse_modified`），可调整输入文本等参数。
- 避免盲目重复操作，综合考虑元素坐标、动作类型与上下文。
- 记录每个操作的成功/失败统计及最后使用时间。

### 2. 多动作步骤验证
- 每个步骤可以包含多个动作（type、click、select 等）。
- EvoUI 会验证**每个动作**是否成功执行。
- 可检测未完成的步骤，例如输入完成但保存按钮未点击。

### 3. 视觉反馈与兜底执行
- 集成 SoM（Semantic Overlay Marking）技术，精准定位 UI 元素。
- 可在页面上标记点击位置，便于调试。
- 如果记忆动作执行失败，会自动触发视觉兜底执行。
- 支持动态修正，当目标元素部分隐藏或页面未完全显示时仍能操作。

### 4. 状态翻转与输入一致性检查
- 检查开关、添加/移除按钮等状态翻转元素。
- 验证输入内容与预期一致，包括密码框等安全字段。

### 5. 自进化与操作治理
- 记录每个操作的历史成功/失败情况。
- AI 根据全局目标和步骤上下文选择最合适的操作。
- 系统通过学习可靠操作，提升自动化成功率。

## 使用示例

```python
from evo_ui import EvoUIOperator

# 初始化操作器
operator = EvoUIOperator(target_url="https://example.com")

# 定义场景步骤
steps = [
    "在输入框中填写项目名称",
    "点击保存按钮创建项目"
]

# 执行场景
for step_idx, step_desc in enumerate(steps):
    # EvoUI 会自动选择或复用操作
    step_success = operator.execute_step(step_desc, global_goal="创建新项目")
    if not step_success:
        print(f"步骤 {step_idx+1} 失败，触发视觉执行兜底。")
```

## 核心优势

* **智能复用**：通过操作记忆和参数化调整，减少重复操作。
* **高鲁棒性**：视觉兜底保证即便记忆动作失败也能完成任务。
* **可追踪**：记录每个步骤的成功/失败、坐标和时间戳。
* **自进化**：系统会随着使用积累经验，学习哪些动作可靠。
